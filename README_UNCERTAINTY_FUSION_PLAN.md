# 不确定性感知融合与动态避障方案（分阶段执行）

> **整体目标**：基于视觉+激光的不确定性感知融合框架，解决动态避障中**速度滞后、转弯丢失、遮挡恢复慢**三大问题，实现快速、安全的动态避障；可选阶段引入 SAC 学习权重/风险调节。

本方案分 4 个阶段，每阶段有明确目标、关键技术与产出，**建议一步一步完成**，先规则版验证再考虑 SAC。

---

## 阶段一：视觉接入与数据对齐（约 3 周）

### 目标
- 把视觉检测/跟踪结果接入系统，与激光数据在**时间、空间**上对齐。
- 转弯时视觉 ID 保持稳定，为后续融合提供“连续性先验”。

### 关键技术
- **YOLOv8**：行人/障碍检测。
- **ByteTrack**：多目标跟踪，输出稳定 ID 与 2D 速度。
- **相机标定**：相机内参、相机 → `base_link` 的 TF。
- **地面投影**：2D 框底边投影到地面，得到 `(x, y)` 与 `base_link` 一致。

### 核心模块（概念）
| 模块 | 输入 | 输出 | 说明 |
|------|------|------|------|
| YOLO 检测节点 | `/front_camera/image_raw` | `/detections` (Detection2DArray) | 行人/障碍 bbox |
| ByteTrack 跟踪节点 | `/detections` | `/tracked_objects` (id, bbox, vx, vy, confidence) | 稳定 ID + 2D 速度 |
| 投影节点 | `/tracked_objects`, `/camera_info` | `/vision_obstacles` (id, x, y, vx, vy, confidence) | 地面坐标，与 DWA 一致 |
| 融合节点 | `/vision_obstacles` + 激光侧障碍（现有 predictor 输出） | `/fused_obstacles` (id, x, y, vx, vy, uncertainty, source) | 最近邻匹配、时间对齐 |

### 产出
- [ ] 在 RViz 中同时看到激光点云与视觉框。
- [ ] 转弯时视觉 ID 保持稳定。
- [ ] 视觉与激光障碍物能匹配上，`/fused_obstacles` 或等价话题可用。

### 当前工程状态
- `bytetrack_ros/`：ByteTrack 桥接节点骨架与真实推理接口已预留；需完成权重下载与话题对接。
- `deptr_mot_ros/`：占位桥接与可视化；可视需要保留或统一到 ByteTrack 方案。
- `sac_dwa_node` 已订阅 `/vision_dynamic_obstacles`，阶段一完成后将此处改为消费融合结果。

## 阶段二：不确定性提取与风险调节（约 4 周）

分两步执行：**先规则版（2a）验证有效，再可选 SAC（2b）**。

### 阶段 2a：规则版不确定性 → 风险调节（约 2 周）

#### 目标
- 从激光预测器与视觉跟踪中提取**可解释的不确定性** `U`。
- 用 `U` **直接映射**到 DWA 的风险项（权重/安全半径/TTC 阈值），不依赖强化学习。

#### 不确定性 `U` 的组成（建议）
- **激光侧**：速度协方差（若用卡尔曼可从 `P[2:4,2:4]` 导出）、簇点数少/回归样本少时的降权。
- **视觉侧**：track 置信度、连续命中/丢失帧数。
- **融合一致性**：同一 id 下视觉与激光位置/速度差异大时，增大 `U`。

#### 风险调节（规则）
- `U` 高时：提高 `weight_predictive`、适当增大 `soft_safety_radius` 或 TTC 惩罚、降低最大线速度/提高急停敏感度。
- `U` 低时：恢复默认权重与半径，允许更激进跟踪目标。

#### 产出
- [ ] 在 `ObstaclePredictor` 或融合节点中暴露 `get_uncertainty(track_id)` 或等价接口。
- [ ] 在 `sac_dwa_node` 或 DWA 调用前，根据 `U` 调整 `planner.set_weights(...)` / 安全半径 / 阈值。
- [ ] 在仿真中定性验证：高不确定性时机器人更保守，低不确定性时更顺畅。

### 阶段 2b：SAC 状态与奖励设计（可选，约 2 周）

#### 目标
- 将 `U` 及融合状态纳入 SAC 的**状态空间**。
- 动作空间：DWA 的权重或安全相关参数。
- 奖励：目标到达、碰撞、平滑性、**不确定性高时靠近障碍的惩罚**。

#### 状态空间（示例）
- 激光速度、激光不确定性、视觉速度、视觉置信度、融合是否匹配、目标相对状态、机器人速度等。

#### 动作空间（示例）
- DWA 四权重 `[w_dir, w_obs, w_vel, w_predictive]` 或安全半径等。

#### 奖励（示例）
- 到达目标 +100；碰撞 -200；平滑性惩罚；**当激光/视觉不确定性高时，按与障碍最小距离扣分**，使策略学会在高 `U` 时自动保守。

#### 产出
- [ ] SAC 环境封装（如 `SACDWAEnv`），与现有 DWA/预测器接口对接。
- [ ] 训练脚本与奖励曲线收敛。
- [ ] 策略在仿真中能复现“高不确定性时提高避障权重”的行为。

## 阶段三：融合决策与实验验证（约 5 周）

### 目标
- 在**三种核心场景**下对比：纯激光 DWA、固定权重融合、规则版不确定性调节、（可选）SAC 融合。
- 用表格与图表证明：速度滞后、转弯丢失、遮挡恢复、整体安全性是否有改善。

### 实验场景与指标（示例）

| 场景 | 配置 | 评价指标 |
|------|------|----------|
| 转弯场景 | 行人约 1 m/s 做 90° 转弯 | 最小距离、碰撞率、反应时间 |
| 遮挡场景 | 行人被挡约 3 s 后重新出现 | ID 切换率、恢复后碰撞率、恢复时间 |
| 密集人群 | 约 5 个行人交叉行走 | ID 切换次数、到达时间、碰撞率 |

### 对比设计（示例）
- **Baseline 1**：纯激光 DWA（当前系统，无视觉）。
- **Baseline 2**：固定权重融合（激光+视觉，无不确定性调节）。
- **方法 1**：规则版不确定性调节（阶段 2a）。
- **方法 2**（可选）：SAC 融合（阶段 2b）。

每个方法在每场景下多次运行（如 50 次），统计：碰撞率、最小距离、ID 切换、到达时间等。

### 产出
- [ ] 实验数据表格与对比图（柱状图/折线图）。
- [ ] 简短分析：本方法在三个问题上相对 baseline 的改进与原因。

- 若采用 SAC：**将卡尔曼/融合不确定性作为 SAC 状态输入，学习在不确定时动态调整避障策略**。

## 工作量与顺序小结

| 阶段 | 时间 | 主要工作 | 建议顺序 |
|------|------|----------|----------|
| 一：视觉接入与对齐 | ~3 周 | YOLO+ByteTrack、标定、投影、融合节点 | 第一步 |
| 二 a：规则版不确定性调节 | ~2 周 | `U` 定义、规则映射到 DWA 风险项 | 第二步 |
| 二 b：SAC 状态与训练 | ~2 周 | 环境封装、奖励、训练与验证 | 可选，在 2a 有效后 |
| 三：实验验证 | ~5 周 | 三场景、多 baseline、表格与图表 | 第三步 |
| 四：论文写作与投稿 | ~3 周 | 写稿、修改、投稿 | 第四步 |

**关键成功因素**
1. **视觉接入要稳**：转弯时 ID 尽量不丢，否则融合与不确定性都会失真。
2. **不确定性要可解释**：`U` 要能反映“速度/关联/遮挡”的不确定程度，便于规则与实验分析。
3. **先规则后 SAC**：先用规则版验证“不确定性 → 风险调节”有效，再上 SAC 学权重，降低训练不稳定风险。
4. **实验要有对比**：至少 3 个 baseline，证明在三个问题上的定量/定性提升。
