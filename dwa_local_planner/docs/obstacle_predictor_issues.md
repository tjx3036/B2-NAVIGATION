## 动态障碍物预测模块当前存在的问题汇总

### 1. 轨迹/障碍物“凭空出现”相关

- 动态障碍物有时在环境中肉眼看不到明显目标的地方突然出现（疑似由激光噪声或反射点聚成的小簇触发）。
- 小规模噪声簇被误当成新轨迹 birth，命中极少帧后即被速度阈值判定为“动态”，在 GUI 中显示为红点。

### 2. 轨迹瞬间消失又重新出现

- 同一动态障碍物的轨迹在 GUI 中会出现“一会儿有、一会儿没”的闪烁现象。
- 短时间内未被任何观测命中的轨迹会被判为死亡并删除，随后在附近位置以新轨迹 ID 重生，造成视觉上的“瞬间消失又凭空出现”。

### 3. 速度估计不稳定 / 跳变

- 部分动态障碍物的速度会在相邻帧间出现从 \(0.x\) m/s 突然跳到几 m/s 甚至更高的现象，然后又快速跌回较小值。
- 在行人交叉或靠近场景中，速度方向和大小会出现明显的非物理跳变（与真实行人运动不符）。

### 4. 预测延迟 / 滞后

- 动态障碍物的轨迹在 GUI 中出现有明显滞后：行人已经开始移动一段距离，预测和动静分离结果才开始显示。
- 在一些情况下，障碍物已经走过去很久，才在 GUI 中被识别并显示为动态。

### 5. 位置预测不准确

- 动态质心（红点）与真实行人所在位置存在可见偏移，尤其在人群较为密集时更明显。
- 预测轨迹（未来点列）在行人转向、减速等非匀速运动时方向和长度与真实路径不匹配。

### 6. 多目标交互下的关联问题

- 多个行人交叉或靠得很近时，轨迹 ID 可能互换，表现为红点直接“跳”到另一个行人身上。
- 在多目标场景（约 20 个行人）下，全局最近邻关联仍可能产生误配，导致轨迹不连续或错绑到错误目标。

### 7. 与 3D 激光特性相关的观测问题

- 3D 激光投影到 2D 后，只保留 \(x, y\) 信息，无法区分高度/形状，部分地面反射或静态结构边缘可能被误判为障碍簇。
- 在遮挡或观测稀疏区域，行人的点云在若干帧内缺失或极少，导致轨迹容易被误删或大幅漂移。
